Política de privacidad

Cómo JW Time gestiona datos locales, conexiones y diagnósticos

Última actualización: 20 de junio de 2026

1. Introducción

JW Time es un proyecto independiente, gratuito y no comercial. Esta política explica qué datos se guardan localmente, qué conexiones a Internet pueden realizarse y qué controles tiene el usuario.

JW Time no recopila ni envía automáticamente al desarrollador datos personales, telemetría, informes de uso, informes de fallos ni registros.

2. Datos guardados localmente

JW Time puede guardar en el equipo configuración de la aplicación, perfiles de congregación, programas, archivos importados, caché de medios, caché de programas, grabaciones de audio si están habilitadas, copias de seguridad y registros técnicos.

Estos datos se usan para ejecutar la aplicación, conservar preferencias, trabajar sin conexión y ayudar al usuario a diagnosticar problemas locales.

3. Datos que no se envían automáticamente

JW Time no envía automáticamente al desarrollador datos personales, estadísticas de uso, telemetría, informes de fallos, registros ni resultados de pruebas de conexión.

Si solicitas ayuda, tú decides si copiar y enviar manualmente registros, capturas de pantalla o resultados de diagnóstico. En ese caso, los datos compartidos se usan solo para brindar soporte.

4. Conexiones a Internet

JW Time puede conectarse a fuentes oficiales para descargar o actualizar programas y medios públicos, al sitio de JW Time para documentación, comunicaciones y archivos de soporte, y a servicios externos solo para funciones explícitas como diagnóstico de red o acceso remoto temporal.

Estas solicitudes sirven para recuperar contenido o comprobar la conectividad. No son telemetría enviada al desarrollador.

5. Servidor web local y acceso remoto

Si se activa, el servidor web de JW Time publica funciones de monitoreo y control en la red local. Los dispositivos de la misma LAN pueden ver datos operativos como el título de la parte, el tiempo restante y el estado del temporizador.

Si el usuario habilita un acceso remoto temporal, el tráfico pasa por el servicio externo necesario para crear ese enlace. El usuario controla la activación y las protecciones disponibles, como PIN o roles.

6. Conservación y eliminación

Los datos permanecen en el equipo del usuario. Algunos archivos temporales, caché o paquetes antiguos se limpian automáticamente cuando la aplicación lo permite; otros pueden eliminarse manualmente desde la configuración o borrando las carpetas locales de la aplicación.

Los archivos locales añadidos manualmente permanecen en la ubicación elegida por el usuario y JW Time no los elimina sin una acción explícita.

7. Derechos del usuario

Como JW Time no mantiene un archivo remoto de los datos del usuario, el acceso, la corrección y la eliminación se realizan principalmente en el propio equipo: desde la configuración, las carpetas locales o desinstalando/eliminando los datos locales.

Para detalles prácticos sobre contenido, medios y caché, consulta también la FAQ.

Para recibir ayuda, puedes contactarnos.
